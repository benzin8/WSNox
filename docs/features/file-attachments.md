# Файлы-вложения

Отправка **любого файла** (документы, архивы, код — что угодно), а не только
фото/видео/аудио. В чате файл показывается карточкой с цветным бейджем
расширения, именем и размером; тап — скачать.

## Бэкенд

- `services/media.py` → `process_file(storage, user_id, file)`: кладёт файл в S3
  **как есть** (без обработки), `msg_type="file"`. Кап `MAX_FILE_BYTES = 50 МБ`
  (под лимит nginx 60M). Meta: `{ filename, ext, size_bytes, content_type }`.
- Имя файла санитайзится (`_safe_filename`): срезаются путь и control-символы,
  длина ≤200. Ключ S3 — UUID, **никогда** не имя файла, поэтому имя — чисто
  отображаемые данные (рендерит React с авто-экранированием).
- `POST /chats/{chat_id}/media`: тип вне image/video/audio-allowlist больше не
  даёт 415 — уходит в `process_file`.
- Безопасность: файлы качаются по presigned-URL на **домене S3**, который
  отделён от домена приложения, поэтому даже залитый html/svg не дотянется до
  cookies/токена приложения.

## Фронтенд

- `AttachmentPicker` принимает любой файл. Маршрутизация: 2+ фото → альбом;
  одно фото/видео → rich-путь (превью-модалка); остальное → карточка файла
  (`onPickFile` → `handleSendMedia` напрямую, `msg_type="file"`).
- `FileCard` — бейдж с расширением, цвет по типу: PDF красный, DOC синий,
  XLS зелёный, PPT оранжевый, ZIP/RAR янтарный, код фиолетовый, аудио розовый,
  остальное — лаймовый бренд. Имя обрезается, размер человекочитаемый
  (`formatBytes`), тап = скачать. Рендерится прозрачно в пузыре (как голосовое),
  цвета текста — по `isOut`.
- Оптимистично карточка видна сразу из локального `File` (имя/размер), после
  ответа — реальные имя/расширение с сервера + presigned-URL.
- Превью в списке чатов и в reply-quote — «📎 Файл» (live + persisted,
  `chat_router` лейблит `last_msg_type == "file"`).

## Вне v1

- Подпись к файлу.
- Несколько файлов за раз (как альбом).
- Превью содержимого (иконки-миниатюры для pdf/docx).
