# Создание каналов

Пользователи могут создавать **публичные каналы**. У канала есть владелец
(создатель), который постит; остальные — подписчики (читают + реагируют).
Вступить можно двумя путями: **поиском по названию** и **по ссылке-приглашению**.

Официальный канал WSNox (singleton, авто-join всех) — отдельная история
([announcements-channel.md](announcements-channel.md)); здесь — про
пользовательские каналы.

## Поведение

### Создание
- Рядом с «Создать группу» — «Создать канал». Модалка: название (обязательно) +
  описание (опционально). Без загрузки аватарки в MVP — авто-аватар как у групп.
- На создание: `Chat(chat_type="channel", name)` + создатель добавляется в
  `chat_members` с ролью **owner** + генерируется `invite_token`.

### Постинг
- Постить может только владелец (роль owner). Остальным канал read-only.
- Владелец постит через обычный инпут (он показан только владельцу).
- WS-путь отправки разрешает постинг в канал, только если автор — owner.

### Вступление
- **Поиск (Б):** поиск находит каналы по названию (ILIKE) → «Подписаться» →
  канал появляется в списке чатов.
- **Ссылка (В):** `/join/<invite_token>` — открыл (залогинен) → подписался →
  канал открылся. У владельца — кнопка «Поделиться ссылкой».
- Подписчик может отписаться; владелец — удалить канал. Показывается счётчик
  подписчиков (`member_count`).

## Данные
- Канал = `Chat(chat_type="channel")` + `chat_members` (role owner/subscriber).
- Новое поле `chats.invite_token` (String, unique, nullable) + Alembic-миграция.
- Поиск каналов — по `name` (ILIKE), только `chat_type="channel"`.

## API
| Метод | Назначение |
|---|---|
| `POST /chats/channels` | создать канал (name, description?) → ChatResponse |
| `POST /chats/channels/{id}/subscribe` | подписаться по id |
| `POST /chats/channels/join/{token}` | подписаться по invite_token → канал |
| `POST /chats/channels/{id}/unsubscribe` | отписаться |
| `GET /chats/search?... ` | поиск расширен: возвращает и каналы по имени |

Реалтайм/посты — через уже существующий WS-путь (owner постит, fan-out по
`chat_members`, как в группах). Реакции в каналах уже работают
([reactions.md](reactions.md)).

## Объём MVP (НЕ берём)
- Загрузка аватарки канала, со-админы (постит только владелец), приватные
  каналы, аналитика, редактирование названия после создания, закреплённые посты.
