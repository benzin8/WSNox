# Реакции на сообщения

Два независимых типа реакций на любое сообщение (текст/медиа/голосовое),
в личке, группах и каналах:

1. **Эмодзи-реакции** — передают эмоцию (👍 ❤️ 😂 😮 😢 🔥).
2. **Усиление ауры** (`⚡`) — энерго-фишка WSNox: сообщение начинает светиться
   тем сильнее, чем больше людей его «зарядили». Social-proof: яркость = число
   людей. Ярче всего раскрывается в группах и каналах.

## Поведение

### Эмодзи
- Набор фиксированный (6 штук), без полного эмодзи-пикера (MVP).
- **Одна эмодзи на человека на сообщение**: тап — поставить, тот же — снять,
  другая — заменить.
- Рисуются чипсами под пузырём: `эмодзи + счётчик`; свой выбор подсвечен.

### Аура
- **Один заряд на человека** (бинарный тумблер). Уровень ауры = число людей.
- Визуал: лаймовое свечение (`box-shadow`) вокруг пузыря; радиус/яркость растут
  с числом зарядов и **насыщаются к ~20** (кап, чтобы не слепить). Рядом чип `⚡ N`.
- При заряде — короткий пульс фонового энерго-орба (через `useEnergy`).
- Эмодзи и аура **независимы**: можно одновременно поставить 😂 и зарядить ауру.

### Где доступно
- Везде. В **каналах** реакции разрешены, хотя постинг там запрещён, — это
  вовлечение аудитории (read-only касается только отправки сообщений).

## Данные

Таблица `message_reactions`:

| поле | тип | назначение |
|---|---|---|
| `id` | PK | |
| `message_id` | FK → messages, on delete cascade | к какому сообщению |
| `user_id` | FK → users, on delete cascade | кто реагирует |
| `reaction_type` | str `'emoji'` \| `'aura'` | тип |
| `emoji` | str \| null | для эмодзи; для ауры — null |
| `created_at` | datetime | |

**Уникальность**: `UNIQUE (message_id, user_id, reaction_type)` — у человека на
сообщение максимум одна эмодзи-реакция и максимум один заряд ауры, независимо.
Смена эмодзи = UPDATE строки. Снятие = DELETE. Alembic-миграция.

## Реалтайм

Действие по WebSocket (как уже есть для send/edit/delete):

```
клиент → WS: { type: "react", message_id, chat_id, react_type, emoji? }
сервер: upsert/delete строку → пересчёт сводки → publish в Redis
Redis pubsub → все воркеры → сокеты участников чата:
  { type: "reaction_update", message_id, chat_id,
    emoji: { "👍": 3, "🔥": 1 }, aura: 5, actor_id }
```

- Клиент обновляет счётчики сообщения; инициатор (`actor_id == me`) подсвечивает
  свой выбор. Кросс-воркерно — как и обычные сообщения (см.
  [push-дедуп](#)/презенс).
- Тумблер оптимистичен на клиенте, сверяется по `reaction_update`.

История сообщений (`GET /chats/{id}/messages`) отдаёт сводку реакций вместе с
сообщениями (счётчики + что выбрал текущий юзер), чтобы при загрузке чата
реакции уже были на месте.
